What is a quantum computer?

A quantum computer is a machine that operates on quantum bits. But what does it mean to be quantum?

Well, it means we are dealing with a probabilistic system in which bits can either be 0, 1 or both in a superposition.

Quantum computing holds a lot of properties:

  • When you try to make an observation, they stop being quantum and return to classical nature
  • They operate on a very very small scale, we can only see quantum nature in very small objects (photons, or electrons for instance)
  • Hold enormous power, 30 qubits = $2^{30}$ calculations
  • Can have multiple subsystems (even classical subsystems)
  • All operations are reversible
  • One operation can change all amplitudes
  • Are able to optimize on some classical algorithms, for instance, factoring large integers: $O (b^3)$ in polynomial time as opposed to non-polynomial time on classical machines
  • Quantum algorithms are formed from a problem that is probabilistic in nature and easy to check

$\newcommand{\ket}[1]{\left|{#1}\right\rangle} \newcommand{\bra}[1]{\left\langle{#1}\right|}$ Perhaps that is enough to wet your appetite. Therefore I shall start from the very beginning.

We can represent a quantum bit with the following: $\begin{matrix} 1 \ 0

\end{matrix}$¶

$\ket{0}$ , $\begin{matrix} 0 \ 1

\end{matrix}$¶

$\ket{1}$

And also in a superposition: $\begin{matrix} 1/\sqrt(2) \\ 1/\sqrt(2) \end{matrix}$ whereby both $\ket{0} or \ket{1}$ are both as likely

As they are probabilistic, the squares of the amplitudes for a given qubit, equal one. $1/\sqrt(2)^2 + 1/\sqrt(2)^2 = 1$

We should also state that amplitudes in quantum bits are represented by complex numbers

Here, we have begun in the $\ket{0}$ state. There is a 100% chance that we will find the $\ket{0}$ because we put it in that state in the first place!

We will begin by applying the hadamard gate. It is a universal quantum logic gate that can put a qubit into a superposition. All quantum gates are unitary and thus are their own inverse. If we applied a hadamard gate twice, we would get what we started with.

First off we have to see where our ket pairs lie within our amplitudes. For instance, given a 2-qubit system, we can see that: $\begin{matrix} 0 & 0 & => \ket{0} Q_0 & \ket{0} Q_1 \\ 0 & 1 & => \ket{0} Q_0 & \ket{1} Q_1 \\ 1 & 0 & => \ket{1} Q_0 & \ket{0} Q_1 \\ 1 & 1 & => \ket{1} Q_0 & \ket{1} Q_1 \end{matrix}$

Applying this to an n-qubit simulation can be difficult.

My solution is the following: We can see here that there is a bit shift after $2^{nqubits-1}$ from $\ket{01}$ to $\ket{10}$

zeroKets = []
oneKets = []

def dertKetPairs(qubit):
    bitShift = pow(2, qubit)
    increaseNum = 0  # num to increase by when we have done 2^qubit+1 calcs
    j = 0  # for incrementing inside loop
    loopAmount = (int(len(amplitudes) / 2))
    for i in range(loopAmount):
        zeroKets.append(j + increaseNum)
        oneKets.append(j + increaseNum + bitShift)
        j = j + 1
        if (j == pow(2, qubit)):
            increaseNum = oneKets[len(oneKets) - 1] + 1
            j = 0
            
    return zeroKets, oneKets


def resetKetPairs():
    zeroKets[:] = []
    oneKets[:] = []


zeroKets, oneKets = dertKetPairs(0)
print(zeroKets)
print(oneKets)

[0, 2]
[1, 3]

Now it's time to apply our hadamard gate.

Our hadamard gate is the following: $1\sqrt(2) \begin{bmatrix} 1 & 1\\ 1 & -1 \end{bmatrix}$
